# Contribuer au projet

Merci de l'intérêt que vous portez à ce projet ! Toutes les contributions (rapports de bugs, corrections, nouvelles fonctionnalités, documentation) sont les bienvenues.

## Comment contribuer ?

### 1. Signaler un bug ou proposer une idée
- Utilisez l'onglet **Issues** de GitHub.
- Soyez le plus précis possible (étapes pour reproduire l'erreur, comportement attendu, système d'exploitation, version de Python, etc.).

### 2. Soumettre du code (Pull Request)
Si vous souhaitez corriger un bug ou rajouter une fonctionnalité :

1. **Forkez** le dépôt.
2. **Clonez** votre fork localement :
   ```bash
   git clone https://github.com/VOTRE_PSEUDO/NOM_DU_DEPOT.git
   ```
3. Créez une **branche** pour votre modification :
   ```bash
   git checkout -b feature/ma-nouvelle-fonctionnalite
   ```
4. Faites vos modifications et testez-les.
5. **Commitez** vos changements avec un message clair (nous utilisons le format des *Conventional Commits*, ex: `feat: ajout de la dictée vocale`, `fix: correction du clic droit`).
6. **Poussez** votre branche sur votre fork :
   ```bash
   git push origin feature/ma-nouvelle-fonctionnalite
   ```
7. Ouvrez une **Pull Request** sur le dépôt original.

## Règles de développement

- **Langue** : Le code (variables, fonctions) peut être en anglais ou en français selon le contexte existant. Assurez-vous simplement de rester cohérent avec le reste du fichier. Les issues et PR peuvent être rédigées en français ou en anglais.
- **Style de code** : Suivez les standards Python classiques (PEP 8). Gardez le code lisible et commenté si nécessaire.

Merci pour votre aide ! 🤝
